char_u *  dst,
int  dstlen

Expand environment variable with path name. "~/" is also expanded, using $HOME. For Unix "~user/" is expanded. Skips over "\ ", "\~" and "\$" (not for Win32 though). If anything fails no expansion is done and dst equals src.

Parameters
srcInput string e.g. "$HOME/vim.hlp"
dstWhere to put the result
dstlenMaximum length of the result

Expand environment variable with path name and escaping. "~/" is also expanded, using $HOME. For Unix "~user/" is expanded. Skips over "\ ", "\~" and "\$" (not for Win32 though). If anything fails no expansion is done and dst equals src. startstr recognize the start of a new name, for '~' expansion.

Parameters
srcpInput string e.g. "$HOME/vim.hlp"
dstWhere to put the result
dstlenMaximum length of the result
escShould we escape spaces in expanded variables?
oneShould we expand more than one '~'?
startstrCommon prefix for paths, can be NULL

Vim's version of getenv(). Special handling of $HOME, $VIM and $VIMRUNTIME, allowing the user to override the vim runtime directory at runtime. Also does ACP to 'enc' conversion for Win32. Results must be freed by the calling function.
